# CODEBASE ARCHITECTURE SNAPSHOT

## 0. Overview
- **Stack**: Node.js monorepo utilizing npm workspaces
  - **Backend (apps/api)**: Express.js, PostgreSQL (via pg Pool), bcryptjs, jsonwebtoken, pdf-parse, Azure Blob Storage SDK (`@azure/storage-blob`)
  - **Frontend (apps/web)**: React (Vite-powered, React Router v6), Tailwind-free Vanilla CSS, React Context state management
  - **Shared package (packages/shared)**: Modular helper module housing logic models (e.g., `computeFit`, `scoreTone`, constants)
- **Entry Points & Config Files**:
  - Root: `package.json`
  - Backend Entry: [server.js](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/api/src/server.js)
  - Backend DB Config: [db.js](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/api/src/config/db.js)
  - Frontend Entry: [main.jsx](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/web/src/main.jsx) -> [App.jsx](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/web/src/App.jsx)
  - Frontend Routing Config: [routes.jsx](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/web/src/config/routes.jsx)
  - Frontend API Config: [api.js](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/web/src/config/api.js)
- **Directory Tree**:
  - `apps/api`: Express.js REST backend
    - `src/config`: Configuration utilities (database setup)
    - `src/db`: Schema migrations and seed scripts
    - `src/middleware`: Custom Express middlewares (JWT auth, error handler)
    - `src/modules`: API features (applications, auth, vacancies)
  - `apps/web`: React SPA frontend
    - `src/config`: API clients and routing configuration
    - `src/middleware`: React contexts (authentication, application state, toast alerts)
    - `src/modules`: Feature page views (appointment, assessment, dashboard, vacancies)
  - `packages/shared`: Common logic and data representations

## A. Database & Schema Layer
The schema uses raw SQL statements managed via migrations and seeds:
- **Entities**:
  - **users**: Accounts for HRMOs and administrators.
    - Fields: `id` (PK, TEXT), `username` (UNIQUE, TEXT), `email` (TEXT), `full_name` (TEXT), `first_name` (TEXT), `last_name` (TEXT), `region` (TEXT), `division` (TEXT), `contact_number` (TEXT), `office` (TEXT), `password_hash` (TEXT), `passcode_hash` (TEXT), `role` (TEXT, default 'hr_officer'), `status` (TEXT, default 'active'), `failed_login_attempts` (INTEGER), `locked_until` (TIMESTAMP), `last_login_at` (TIMESTAMP), `password_changed_at` (TIMESTAMP), `must_change_password` (BOOLEAN), `created_at` (TIMESTAMP), `updated_at` (TIMESTAMP)
  - **positions**: Qualification requirements mapping for roles.
    - Fields: `id` (PK, TEXT), `title` (TEXT), `track` (TEXT), `required_bachelor_degree` (TEXT), `required_degree_keywords` (TEXT), `min_years_experience` (INTEGER), `min_training_hours` (INTEGER), `eligibility_required` (TEXT), `created_at` (TIMESTAMP), `updated_at` (TIMESTAMP)
  - **job_clusters**: Groups positions by location scope.
    - Fields: `id` (PK, TEXT), `position_id` (TEXT, FK positions.id), `division` (TEXT), `region` (TEXT), `created_at` (TIMESTAMP)
  - **vacancies**: Available plantilla openings.
    - Fields: `id` (PK, TEXT), `position_id` (TEXT, FK positions.id), `item_no` (UNIQUE, TEXT), `title` (TEXT), `school` (TEXT), `division` (TEXT), `region` (TEXT), `status` (TEXT, default 'open'), `filling_up_status` (VARCHAR(50), default 'UNFILLED'), `job_cluster_id` (TEXT, FK job_clusters.id), `school_level` (TEXT), `school_id` (INTEGER), `posting_start` (TIMESTAMP), `posting_end` (TIMESTAMP), `salary_grade` (INTEGER), `created_at` (TIMESTAMP), `updated_at` (TIMESTAMP)
  - **applicants**: High-density profile storage for applicants.
    - Fields: `id` (PK, SERIAL), `password_hash` (VARCHAR(255)), `surname` (VARCHAR(100)), `first_name` (VARCHAR(100)), `middle_name` (VARCHAR(100)), `date_of_birth` (DATE), `place_of_birth` (VARCHAR(255)), `sex` (VARCHAR(20)), `civil_status` (VARCHAR(50)), `citizenship` (VARCHAR(100)), `blood_type` (VARCHAR(10)), `gsis_id_no` (VARCHAR(50)), `pag_ibig_id_no` (VARCHAR(50)), `philhealth_no` (VARCHAR(50)), `sss_no` (VARCHAR(50)), `residential_address` (TEXT), `permanent_address` (TEXT), `telephone_no` (VARCHAR(50)), `mobile_no` (VARCHAR(50)), `email_address` (UNIQUE, VARCHAR(150)), `spouse_surname` (VARCHAR(100)), `spouse_first_name` (VARCHAR(100)), `spouse_occupation` (VARCHAR(150)), `father_surname` (VARCHAR(100)), `father_first_name` (VARCHAR(100)), `mother_maiden_surname` (VARCHAR(100)), `name_extension` (VARCHAR(50)), `height` (VARCHAR(20)), `weight` (VARCHAR(20)), `agency_employee_no` (VARCHAR(50)), `citizenship_type` (VARCHAR(50)), `spouse_middle_name` (VARCHAR(100)), `spouse_name_extension` (VARCHAR(50)), `spouse_employer_business` (VARCHAR(150)), `spouse_business_address` (VARCHAR(255)), `spouse_telephone` (VARCHAR(50)), `father_middle_name` (VARCHAR(100)), `father_name_extension` (VARCHAR(50)), `mother_first_name` (VARCHAR(100)), `mother_middle_name` (VARCHAR(100)), `children_details` (JSONB), `family_background` (JSONB), `educational_background` (JSONB), `civil_service_eligibility` (JSONB), `work_experience` (JSONB), `voluntary_work` (JSONB), `learning_and_development` (JSONB), `other_information` (JSONB), `questionnaire_responses` (JSONB), `name` (VARCHAR(255)), `code` (VARCHAR(255)), `local_resident` (BOOLEAN), `bachelor_degree` (VARCHAR(255)), `major` (VARCHAR(255)), `years_experience` (DOUBLE PRECISION), `training_hours` (DOUBLE PRECISION), `eligibility` (VARCHAR(255)), `applicant_number` (VARCHAR(255)), `created_at` (TIMESTAMP WITH TIME ZONE), `updated_at` (TIMESTAMP WITH TIME ZONE)
  - **applications**: Links applicants to job clusters.
    - Fields: `id` (PK, TEXT), `application_number` (UNIQUE, TEXT), `job_cluster_id` (TEXT, FK job_clusters.id), `applicant_id` (INTEGER, FK applicants.id), `status` (TEXT, default 'Application Submitted'), `application_status` (TEXT, default 'Application Submitted'), `date_applied` (TIMESTAMP), `documents` (TEXT), `documentary_complete` (BOOLEAN), `doc_checklist` (TEXT), `reason` (TEXT), `assessment_status` (TEXT), `comparative_assessment_scores` (TEXT), `appointment_status` (TEXT), `appointment_date` (TIMESTAMP), `appointment_item_no` (TEXT), `appointment_reference_code` (TEXT), `created_at` (TIMESTAMP), `updated_at` (TIMESTAMP)
  - **application_history**: Audit trail for state modifications.
    - Fields: `id` (PK, TEXT), `application_id` (TEXT, FK applications.id), `at` (TIMESTAMP), `text` (TEXT)
  - **qual_evals**: Qualification standards checklist.
    - Fields: `id` (PK, TEXT), `application_id` (TEXT, FK applications.id), `result` (TEXT), `overall_fit` (DOUBLE PRECISION), `degree_score` (DOUBLE PRECISION), `experience_score` (DOUBLE PRECISION), `training_score` (DOUBLE PRECISION), `eligibility_score` (DOUBLE PRECISION), `degree_decision` (TEXT), `experience_decision` (TEXT), `training_decision` (TEXT), `eligibility_decision` (TEXT), `documentary_complete` (BOOLEAN), `remarks` (TEXT), `area_scores` (TEXT), `at` (TIMESTAMP)
  - **notifications**: User alerts.
    - Fields: `id` (PK, TEXT), `user_id` (INTEGER, FK applicants.id), `title` (TEXT), `message` (TEXT), `at` (TIMESTAMP)
  - **agap_schools** *(Not defined in seeds but queried)*: Contains division schools mapping.
    - Fields: `region` (TEXT), `division` (TEXT), `school_id` (INTEGER), `school_name` (TEXT)
- **Relationships**:
  - `job_clusters.position_id` -> `positions.id` (1:N, On Delete: Cascade)
  - `vacancies.position_id` -> `positions.id` (1:N, On Delete: Cascade)
  - `vacancies.job_cluster_id` -> `job_clusters.id` (1:N, On Delete: Set Null)
  - `applications.job_cluster_id` -> `job_clusters.id` (1:N, On Delete: Cascade)
  - `applications.applicant_id` -> `applicants.id` (1:N, On Delete: Cascade)
  - `application_history.application_id` -> `applications.id` (1:N, On Delete: Cascade)
  - `qual_evals.application_id` -> `applications.id` (1:N, On Delete: Cascade)
  - `notifications.user_id` -> `applicants.id` (1:N, On Delete: Cascade)
- **Schema Control**: Database seed is loaded dynamically via `seed.js`. Migrations are located in [migration_cluster.js](file:///c:/InsightED%20and%20STRIDE/InsightED%20v2/agap_portal/apps/api/src/db/migration_cluster.js).

## F. Gaps, Risks & Unknowns
- **Pre-existing Table Dependency (`agap_schools`)**: Referenced in `verify-passcode` / `autocompleteSchools` queries, but dropped/created dynamically outside core `seed.js`.
- **Azure Storage Connection String Configuration**: If the env key `AZURE_STORAGE_CONNECTION_STRING` is missing, files fail gracefully to standard PDF buffers. Local deployments have `existsInAzure` checks mapped to `false`.
- **Evaluation Locking Logic**: Evaluation locking features inside `reviewApplication` are declared as disabled (`const app = null;` override), keeping evaluations editable.
- **Auto-Expired Vacancies Task**: Vacancies checking and expiration triggers run inline upon `/api/vacancies` GET requests instead of decoupled background cron jobs.
